从 CSS Transition/Animation 到 JavaScript 动画，再到新一代 Web Animations API，系统掌握前端动画的完整技术栈

前言

前端动画不是”锦上添花”——好的动画可以提升用户体验、引导注意力、让交互更自然。

本文将系统梳理前端动画的三大技术路线，帮你根据场景选择最合适的方案。

六、性能对比

方案 性能 控制力 适用场景
CSS Transition ⭐⭐⭐⭐⭐ ⭐⭐ 简单的状态切换
CSS Animation ⭐⭐⭐⭐⭐ ⭐⭐⭐ 预定义的连续动画
WAAPI ⭐⭐⭐⭐⭐ ⭐⭐⭐⭐ 需 JS 控制的高性能动画
requestAnimationFrame ⭐⭐⭐⭐ ⭐⭐⭐⭐⭐ 自定义缓动、逐帧控制
GSAP/Framer Motion ⭐⭐⭐⭐ ⭐⭐⭐⭐⭐ 复杂动画编排

七、总结

  1. 简单的交互反馈 → CSS Transition
  2. 预定义的装饰性动画 → CSS Animation
  3. 需要 JS 控制路由切换等 → WAAPI
  4. 复杂的时序编排、滚动动画 → GSAP / Framer Motion
  5. 核心原则：始终对 transform + opacity 做动画
