JavaScript 设计模式在前端的应用

发表于 2026-03-08 分类于 JavaScript 阅读次数：
掌握 8 种前端最常用的设计模式：单例模式、观察者模式、代理模式、工厂模式等，含实际业务场景代码

前言

设计模式是经过验证的解决方案，而非教条。本文挑选前端开发中最常用的 8 种模式，每种都配合实际业务场景，不讲空泛的理论。

一、单例模式（Singleton）

确保一个类只有一个实例，并提供全局访问点。

场景：全局状态管理（如 Pinia/Vuex 的 store）

场景：全局弹窗 / Toast

二、观察者模式（Observer）

一对多依赖：当一个对象状态变化时，所有依赖它的对象都得到通知。

场景：Vue 的响应式系统

发布订阅模式 vs 观察者模式：

  • 观察者模式：Subject 直接通知 Observer
  • 发布订阅模式：通过 Event Bus 解耦（如 mitt）

三、代理模式（Proxy）

为另一个对象提供一个替身以控制对它的访问。

场景：图片懒加载

场景：缓存代理

四、工厂模式（Factory）

用一个工厂函数代替 new 创建对象，将创建逻辑封装。

场景：创建不同类型的弹窗

五、策略模式（Strategy）

定义一系列算法，把它们封装起来并可互相替换。

场景：表单校验

六、装饰器模式（Decorator）

动态地为对象添加行为，不改变原有代码。

场景：权限控制

七、迭代器模式（Iterator）

提供一种顺序访问集合元素的方法，而不暴露其内部结构。

场景：分页加载

八、模块模式（Module）

将私有变量和函数封装在闭包中，只暴露公共接口。

总结
