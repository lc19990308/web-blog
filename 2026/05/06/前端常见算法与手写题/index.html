发表于 2026-05-06 分类于 JavaScript 阅读次数：
前端面试和日常开发中最常见的算法与手写题：防抖/节流、深拷贝、数组去重、Promise 系列、发布订阅模式等，附完整实现与测试用例

前言

前端面试中的手写题，考察的不是算法竞赛级别的能力，而是对 JavaScript 语言特性和编程思维的理解。

本文整理了 10 道最高频的手写题，每道都包含：实现思路 → 完整代码 → 测试用例 → 考察要点。

一、防抖（Debounce）

场景： 搜索输入框，用户停止输入 300ms 后才发请求。

考察要点： this 指向、闭包、setTimeout、apply/call。

二、节流（Throttle）

场景： 滚动事件、拖拽，固定频率触发。

防抖 vs 节流：

场景 方案 说明
搜索输入 防抖 停止输入后才触发
滚动/拖拽 节流 固定频率触发
按钮点击 防抖 防止重复提交

三、深拷贝（Deep Clone）

为什么不直接用 JSON.parse(JSON.stringify(obj))？

  • 不支持函数、undefined、Symbol、BigInt
  • 不支持 Date（变为字符串）
  • 不支持循环引用
