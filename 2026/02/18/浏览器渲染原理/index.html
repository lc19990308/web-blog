浏览器渲染原理：从 URL 到像素

发表于 2026-02-18 分类于 浏览器 阅读次数：
理解浏览器从输入 URL 到页面渲染的完整过程：关键渲染路径、DOM/CSSOM 构建、布局与绘制详解

前言

打开一个网页，浏览器背后做了大量工作。理解关键渲染路径（Critical Rendering Path）是前端性能优化的基础。

整个过程分为 6 步：

一、导航阶段

1.1 DNS 解析

每次握手约 1 个 RTT（往返时间）。HTTP/2 多路复用减少连接数。

二、解析阶段

2.1 构建 DOM 树

2.2 构建 CSSOM 树

CSS 解析不会阻塞 DOM 构建，但会阻塞渲染——因为浏览器不知道样式前不会渲染。

2.3 JavaScript 阻塞

属性 下载 执行时机
无 遇到时 立即执行，阻塞解析
async 异步下载 下载完立即执行
defer 异步下载 DOM 解析完后执行

推荐： <script defer> 放在 <head> 中。

三、渲染阶段

3.1 构建渲染树

3.2 布局（Layout）

计算每个可见元素在视口中的几何位置（宽、高、x、y）：

触发重排（Reflow）的操作：

  • 修改 width / height / margin / padding
  • 修改 display: none → block
  • 读取 offsetHeight / offsetWidth / scrollTop

3.3 绘制（Paint）

将元素的像素填充到屏幕上——文本、颜色、图片、边框、阴影。

触发重绘（Repaint）的操作：

  • 修改 color / background-color / visibility
  • 修改 box-shadow / outline

四、重排 vs 重绘 vs 合成

操作 成本 触发条件
重排（Reflow） 最高 修改几何属性（宽高、位置）
重绘（Repaint） 中 修改外观属性（颜色、背景）
合成（Composite） 最低 transform / opacity
