前端安全防护指南：XSS、CSRF 与 HTTPS

发表于 2026-02-10 分类于 安全 阅读次数：
前端开发中必须了解的安全知识：XSS 攻击与防御、CSRF 原理与 Token 机制、HTTPS 与 CSP 内容安全策略

前言

前端安全不是后端专属的责任。XSS 攻击、CSRF 伪造请求、敏感信息泄露——这些安全威胁如果不了解，你的项目随时可能被攻击。

本文聚焦前端能做的安全防护，不讲空泛的概念，直接给可落地的方案。

一、XSS（跨站脚本攻击）

1.1 什么是 XSS

攻击者在网页中注入恶意脚本，当用户访问时执行该脚本，从而窃取 cookie、token 等信息。

三种类型：

类型 原理 危害
存储型 恶意代码存到服务器（如评论区） 最大（持久化）
反射型 恶意代码在 URL 参数中 中（需诱导点击）
DOM 型 通过前端 JS 动态注入 中

方案 2：设置 Content-Security-Policy

方案 3：HttpOnly Cookie

二、CSRF（跨站请求伪造）

2.1 什么是 CSRF

用户登录了银行网站 A，然后访问了恶意网站 B。B 网站构造了一个请求（转账），因为浏览器自动携带了 A 网站的 cookie，请求看起来是”合法”的。

2.2 防御方案

方案 1：SameSite Cookie（最推荐）

方案 2：CSRF Token

方案 3：验证 Referer/Origin

五、安全头速查

头部 作用 推荐值
Content-Security-Policy 限制资源来源 default-src 'self'
X-Content-Type-Options 禁止 MIME 嗅探 nosniff
X-Frame-Options 防止点击劫持 DENY 或 SAMEORIGIN
Strict-Transport-Security 强制 HTTPS max-age=31536000
Referrer-Policy 控制 referer 信息 strict-origin-when-cross-origin
