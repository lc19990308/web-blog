Promise 完全指南：从原理到实战

发表于 2021-08-13 分类于 JavaScript 阅读次数：
从 Promise 的核心状态机到手写实现，再到业务中的并发控制、请求重试、超时处理等实战场景，一篇彻底搞懂 Promise

前言

Promise 是现代 JavaScript 异步编程的基石。无论你是用 async/await 还是回调，底层都离不开 Promise。很多同学会用 .then() 和 .catch()，但在遇到并发控制、请求重试、超时中断等场景时却无从下手——因为对 Promise 的理解停留在”会用”层面。

本文的目标是帮你从 “会用” 进阶到 **”懂原理、能实战”**。

一、核心概念：Promise 状态机

1.1 三个状态

Promise 本质上是一个状态机，只有 3 种状态：

两个关键特性：

  1. 状态一旦改变就凝固（称为 settled/resolved），不会再次变化
  2. 状态变更不可逆：Pending → Fulfilled 后，无法再变为 Rejected

1.2 Promise 的执行时机

关键理解：Promise 构造函数中的代码是同步执行的，只有 .then()/.catch() 的回调是微任务，在当前宏任务结束后、下一个宏任务前执行。

三、常用 API 详解与对比

3.1 静态方法全景

方法 用途 成功条件 失败条件
Promise.all() 全部成功 所有成员 resolved 任一 rejected
Promise.race() 竞速 最快 resolved/rejected 最快 resolved/rejected
Promise.allSettled() 全部完成 所有成员 settled（不论成败） 永不 rejected
Promise.any() 任意成功 最快 resolved 全部 rejected
Promise.resolve() 包装为 resolved 总是成功 永不失败
Promise.reject() 包装为 rejected 永不成功 总是失败

3.2 Promise.all 并发请求

注意：Promise.all 是”快速失败”（fail-fast）策略——一个失败就整体失败。

五、async/await 本质

async/await 是 Promise 的语法糖，本质还是 Promise。

async/await 的优势：

方面 Promise 链 async/await
可读性 链式回调，复杂逻辑难读 同步代码风格
错误处理 .catch() 链 try/catch 块
条件分支 需要 if/else + return 直接 if/else
循环 需要递归或 reduce for 循环直接 await

七、方案选型对比

场景 推荐方案 理由
多个无依赖请求 Promise.all() 并行，全部拿到才继续
请求间有依赖（A→B→C） async/await 链式 天然串行，代码清晰
只需最快结果 Promise.race() 竞速模式
多个请求必须全部完成 Promise.allSettled() 不中断，收集所有结果
多个备用源，取最快成功的 Promise.any() 一个成功就够
控制并发数 concurrentControl() 工具函数 防止资源耗尽
请求超时 Promise.race() + 超时 Promise 优雅降级
失败自动重试 fetchWithRetry() 工具函数 提高稳定性

八、Promise 与事件循环（Event Loop）

执行顺序规则：

  1. 同步代码（当前宏任务）
  2. 清空微任务队列（Promise.then、MutationObserver）
  3. 执行下一个宏任务（setTimeout、setInterval、I/O）
