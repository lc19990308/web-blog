map 和 foreach 的区别

众所周知，map 会返回一个新数组，map 可以 return 而 fo rEach 却不行。除了这些 map 和 foreach 还有那些区别吗？

1.map 没有副作用
这里的副作用指的是，map 设计的时候更适合对数组进行遍历，返回一个新的数组。map，设计的时候不希望我们遍历数组成员的时候，对外部的变量，dom 进行操作，而 foreach 更适合遍历的时候，去修改遍历。

2.foreach 遍历数组成员的时候，可以对每个成员的数据进行修改。这就是 foreach 的副作用。

filter

对数组进行过滤

隐式调用的时候，不用 return

flat

可以把数组里面的数组给排扁,flat 填写要拍扁的数组层数，Infinity 是无限大，如果不确定要拍扁多少层，要不拍扁的话，使用 Infinity。Infinity 是数字类型。

flatmap

flat 把数组拍扁，map 是遍历返回新的数组。

reduce

reduce 是一个累加器，他会遍历数组的过程当中，不断积累数组，并且进行统计。

比如我们有一个商品列表，我需要计算里面的总价格，总数量。这个时候使用累加器就很合适。

注意：累加器必须使用 return，并且有一个初始值，初始值可以是任意数据类型

find 与 findIndex

find 和 findIndex 使用方式一样，只不过一个找的是数组成员一个找的是下标位置。

注意：find 和 findIndex 只会寻找第一个符合条件的成员，filter 不同，他会寻找所有如何条件的，过滤掉不符合的

some 和 every

some 当数组成员有一个满足条件，some 就会返回 true，every 需要数组成员全部满足条件才返回 true。

some 和 includes

some 适合查找复杂的数组成员，includes 只适合简单的数组成员。

slice 与 splice

slice 对数组成员进行切片，然后返回一个新的数组,不改变原有的数组。

splice 对数组的某个成员进行删除
注意：第一个参数是开始的下标位置，第二个是要删除的数量，第三个是要插入的数据

如果 splice 只传第一个参数的话，那就会把下标位置后面的数据全部清空

sort 对数据进行排序

sort 如果不传排序规则，会默认把数据转为字符串 按照 unicode 进行排序

join 把数组转为字符串

concat 把数组合并为一个数组

reverse

reverse 使数组反转

shift 和 pop

一个删除数组第一个成员，一个是最后一个。

fill

用于填充数组，给数组默认值，第一个参数要填充的值，第二第三个参数，要填充的位置。

关于 js 的副作用

修改全局对象，外部变量，对 localStorage 操作 和读写文件，改变 dom，发网络请求，都是副作用。
