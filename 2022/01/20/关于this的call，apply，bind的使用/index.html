JavaScript 中的 this 指向：call、apply、bind 完全解析

发表于 2022-01-20 分类于 JavaScript 阅读次数：
彻底搞懂 JavaScript 的 this 指向规则，以及 call、apply、bind 三种显式绑定方式的区别与实战应用

前言

this 是 JavaScript 中最容易让人困惑的概念之一。它的指向不是在定义时确定的，而是在调用时确定的——谁调用了它，它就指向谁。

优先级：new > 显式 > 隐式 > 默认

二、call vs apply vs bind

2.1 call——立即执行，参数列表

2.2 apply——立即执行，参数数组

2.3 bind——不执行，返回新函数

2.4 三者对比

方法 是否立即执行 参数传递 返回值 适用场景
call ✅ 立即执行 列表 a, b, c 函数结果 需要立即调用、参数少
apply ✅ 立即执行 数组 [a, b] 函数结果 参数是数组
bind ❌ 不执行 列表 a, b, c 新函数 回调、事件监听

四、箭头函数与 this

箭头函数没有自己的 this，它继承外层作用域的 this：

特性 普通函数 箭头函数
this 调用时确定 定义时确定（继承外层）
call/apply/bind 可以改变 this 无法改变
作为构造函数 ✅ ❌
arguments ✅ ❌（需用 rest）
适用场景 对象方法、构造函数 回调、事件监听
